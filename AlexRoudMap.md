## 📚 Используемые технологии и инструменты

### Языки программирования
- Python 3.11+ — основной язык backend разработки
- PyPy 7.3+ — JIT-компилятор Python (использовать только если Никита подтвердит совместимость на Неделе 1)
- SQL — язык запросов для работы с БД

### Фреймворки и библиотеки

#### ORM и база данных
- SQLAlchemy 2.0+ — ORM для работы с БД через Python объекты
- Alembic — управление миграциями схемы БД
- SQLite 3 — встроенная реляционная БД

#### Валидация данных
- Pydantic v2 — валидация, типизация и сериализация данных

#### Работа с данными
- csv (встроенный) — импорт/экспорт CSV
- json (встроенный) — работа с JSON
- python-dateutil — работа с датами
- pykeepass — импорт из KeePass (.kdbx) (**вынесен в Спринт 7+, не блокирует основной план**)

#### Утилиты
- secrets (встроенный) — генерация случайных чисел для паролей
- pathlib (встроенный) — работа с путями файлов
- shutil (встроенный) — операции с файлами

### Тестирование и качество кода
- pytest 7.4+ — фреймворк для тестирования
- pytest-cov 4.1+ — покрытие кода тестами
- factory_boy — генерация тестовых данных
- faker — генерация реалистичных данных
- black 23.12+ — автоформатирование кода
- flake8 6.1+ — линтер Python
- mypy 1.7+ — статическая типизация
- isort — сортировка импортов

### Инструменты разработки
- Git — контроль версий
- GitHub — хостинг репозитория и Code Review
- GitHub Actions — CI/CD pipeline
- VS Code / PyCharm — IDE
- DB Browser for SQLite — визуальный просмотр БД
- Postman / Insomnia — тестирование API

### Документация
- Markdown — документация проекта
- draw.io — диаграммы БД (ER-diagrams)

## ⚠️ Важные напоминания

1. **Всегда шифруй чувствительные данные** — используй EncryptionHelper
2. **Не пиши криптографию сам** — только через модули Никиты
3. **SQLAlchemy ORM защищает от SQL injection** — не пиши raw SQL с пользовательскими данными
4. **Pydantic валидирует ВСЕ входящие данные** — без исключений
5. **Двойное подтверждение для удаления** — `confirmed=True` обязательно
6. **Не логируй пароли** — даже зашифрованные, даже в debug режиме
7. **Тестируй граничные случаи** — пустые поля, Unicode, длинные строки
8. **Задавай вопросы Никите сразу** — не гадай в вопросах безопасности
